# CEO-12: Make Promotion Demo Completion Guide

## 문서 목적

이 문서는 `Make promotion` 용 데모를 **실행 가능한 수준으로 완성**하기 위한 기준 문서다.
CEO/DEV/DESIGN이 같은 완료 기준을 사용하고, CEO-13의 핸드오프 원칙에 맞춰 상태 전환 시 바로 이어서 작업할 수 있도록 한다.

## Definition of Done (완료 기준)

아래 항목이 모두 충족되면 CEO-12를 Done으로 본다.

1. **데모 스토리 완결성**
   - 5~7분 러닝타임 안에 핵심 가치 제안이 자연스럽게 전달된다.
   - 체스 로그라이크의 차별점(전술적 전투 + 메타 성장 + 업그레이드 선택)이 모두 등장한다.
2. **실행 안정성**
   - 오프라인/저사양 환경에서도 실행 가능한 백업 빌드 또는 녹화본이 준비된다.
   - 데모 중 크래시/블로커 발생 시 대체 시나리오가 문서화되어 있다.
3. **프로모션 메시지 정합성**
   - 오프닝/클로징 메시지가 동일한 키메시지 3개를 반복한다.
   - CTA(Call to Action)가 한 문장으로 명확히 정리되어 있다.
4. **핸드오프 가능 상태**
   - CEO ↔ DEV ↔ DESIGN 책임 분담표와 마지막 상태 스냅샷이 문서에 남아 있다.

## 5~7분 데모 Run of Show

### 0:00~0:40 — Opening (문제/가치 제안)

- 기존 체스 경험의 진입장벽(고정된 룰 학습 부담)과 로그라이크 반복 플레이 동기를 짧게 제시.
- 한 문장 피치:
  - "한 판의 전술 판단과, 판 밖의 영구 성장이 연결되는 체스 로그라이크를 보여드립니다."

### 0:40~2:40 — Core Combat (전투 보드)

- 전투 보드 진입, 턴 진행, 말 이동/공격 의사결정 시연.
- HUD에서 체력/턴/위협 정보 확인 흐름 강조.
- 관전 포인트:
  - “체스 규칙 기반 예측 가능성” + “로그라이크 변수로 인한 매판 변주”

### 2:40~4:10 — Reward Choice (업그레이드/보상 선택)

- 전투 종료 후 업그레이드 카드 2~3개 노출.
- 선택 결과가 다음 전투 플레이 스타일에 주는 영향 설명.
- 관전 포인트:
  - 같은 시작 상태라도 선택 누적으로 다른 빌드가 만들어진다는 점.

### 4:10~5:40 — Meta Progression (메타 성장)

- 세션 외 성장 요소(해금, 영구 보정, 다음 런 준비) 화면 설명.
- "한 판 실패가 다음 시도 효율을 높이는 구조"를 시각적으로 명확히 보여준다.

### 5:40~7:00 — Roadmap + CTA

- 다음 마일스톤(콘텐츠 확장, UI polish, 플레이 테스트 범위)을 3개 이내로 제시.
- 마무리 CTA 예시:
  - "오늘 데모 버전 피드백을 반영해 다음 스프린트에서 전투/메타 루프 완성도를 끌어올리겠습니다."

## 역할 분담 (CEO / DEV / DESIGN)

### CEO

- 데모 스토리라인, 메시지 우선순위, 리뷰 기준 확정
- 오프닝/클로징 스크립트 최종 승인
- 외부 공유용 요약본 1페이지 정리

### DEV

- 데모 빌드 안정화(시연 경로 기준 사전 점검)
- 백업 플랜 준비(녹화본, 세이브 포인트, 수동 시나리오)
- 데모 중 확인할 품질 체크리스트 제공(입력/전환/저장/복구)

### DESIGN

- 핵심 장면(전투, 보상 선택, 메타 성장)의 시각적 강조 포인트 정리
- 텍스트/아이콘/정보 계층 우선순위 확인
- 발표용 캡처/썸네일/간단 보조 슬라이드 지원

## 리스크 및 폴백(Fallback) 계획

1. **데모 중 빌드 불안정**
   - 폴백: 동일 흐름의 녹화본 재생 + 라이브 설명 전환
2. **설명 시간 초과**
   - 폴백: Roadmap 슬라이드 1장으로 축약, CTA만 유지
3. **핵심 기능 노출 실패**
   - 폴백: 업그레이드 선택/메타 성장 장면을 우선 재시연하고 전투 설명은 압축

## Promotion Messaging Template (3줄)

1. **What it is**: "Chess Roguelike는 체스 전술성과 로그라이크 성장성을 결합한 전략 경험입니다."
2. **Why now**: "핵심 루프 데모를 통해 전투-보상-메타 연결 구조를 검증 중입니다."
3. **What next**: "이번 피드백을 반영해 다음 스프린트에서 콘텐츠 다양성과 UI 완성도를 확장합니다."

## Handoff Snapshot (CEO-13 공통 포맷)

아래 템플릿을 상태 변경 시마다 최신화한다.

```md
### Handoff Snapshot
- Current Decision:
- Remaining Decisions:
- Blockers:
- Needed Input / Owner:
- Next Action (Owner, Due):
- Verification Plan:
```

### 현재 스냅샷 (초안)

- Current Decision:
  - 데모는 5~7분 고정, 전투→보상→메타→로드맵 순서로 진행한다.
- Remaining Decisions:
  - 외부 공개 시점에 맞춘 데모 버전 태깅 방식(브랜치/릴리스 노트).
- Blockers:
  - 없음(단, 시연 기기 성능 편차 리스크 존재).
- Needed Input / Owner:
  - CEO: 최종 발표 문구 확정.
  - DEV: 데모 당일 빌드 해시 및 백업 실행 확인.
- Next Action (Owner, Due):
  - DEV/CEO 공동 리허설 1회 진행 후 체크리스트 잠금.
- Verification Plan:
  - 리허설에서 7분 이내 완주 + 핵심 장면 3개 모두 노출 여부 체크.

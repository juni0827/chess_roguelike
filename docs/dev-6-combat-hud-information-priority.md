# DEV-6: 체스 로그라이크 전투 HUD와 정보 우선순위 설계

## 1) Goal

전투 중 플레이어가 **지금 무엇을 해야 하는지 2초 안에 판단**할 수 있도록 HUD를 설계한다.

- 보드 조작 맥락에서 필수 정보(P0)는 항상 고정 노출한다.
- 선택/결과/보조 정보(P1~P3)는 상태에 따라 노출 우선순위를 바꾼다.
- DEV-9(보드 UI/입력 처리), DEV-8(상태 머신), DEV-10(데이터 구조)로 바로 연결 가능한 컴포넌트 단위 규칙을 제공한다.

---

## 2) Inputs

### 제품/요구사항 전제

- 플랫폼: Android 우선 (세로 레이아웃 기준)
- 코어 루프: 메인 메뉴 → 전투 → 업그레이드 선택 → 다음 전투
- 현재 코드 기준 연결 대상
  - 화면: `apps/android/src/main/res/layout/activity_game.xml`
  - 상태/액션: `GameState`, `GameAction`, `GameEvent`
  - 입력: `BoardView` square/piece selection

### UX 원칙

1. **상황 인지 우선**: 턴/목표/승패 위험도는 항상 상단 고정.
2. **행동 유도 명확화**: 현재 가능한 행동은 하단 액션 영역에 집중.
3. **인지 부하 절감**: 비핵심 통계는 축약/접기 허용.
4. **상태 전환 일관성**: 전투, 능력 부여 선택, 결과 모달 간 정보 점프 최소화.

---

## 3) Screens (대상 화면)

## S1. 기본 전투 HUD (Default Combat)

- 목적: 플레이 중 상시 노출되는 기본 상태.
- 구성
  - 상단: 라운드, 턴 상태, 현재 목표, 진행도
  - 중앙: 체스 보드 + 선택/이동 가능 하이라이트
  - 하단: 아군/적군 수, 포획 수, 즉시 행동 버튼

## S2. 능력 부여 대상 선택 HUD (Target Selection)

- 트리거: 능력형 업그레이드 선택 직후
- 차이점
  - 상단 목표 텍스트를 "능력 부여할 아군 기물 선택"으로 강제 오버라이드
  - 하단에 "취소 불가/선택 필요" 안내를 P1로 표시
  - 유효하지 않은 선택(적군, 킹, 빈 칸) 시 P0 경고 토스트

## S3. 적 턴 관전 HUD (Enemy Turn Watch)

- 트리거: `TurnState.ENEMY_TURN`
- 차이점
  - 액션 버튼 최소화(불필요 버튼 숨김)
  - 상단에 "적 반격 예상" 목표 문구 우선 노출
  - 보조 통계(P3)는 축약 텍스트로만 유지

## S4. 전투 종료 결과 모달 (Victory / Game Over)

- 트리거: `GameEvent.RunEnded`
- 차이점
  - 보드 상호작용 잠금
  - 결과 모달에서 점수/보상/다음 행동(재시작/메인 메뉴) 제공
  - HUD 배경 정보는 보조화(시각 강조 낮춤)

## 4) 정보 우선순위 (P0~P3)

## P0: 절대 고정 (항상 노출, 숨김 금지)

- 턴 상태 (`PLAYER_TURN`, `PLAYER_DOUBLE_MOVE`, `ENEMY_TURN`, `ROUND_WON`, `GAME_OVER`)
- 현재 목표 문구(행동 지시)
- 라운드 진행(현재/최대, progress)
- 치명 상태 알림(능력 대상 미선택, 패배/승리, 유효하지 않은 선택)

## P1: 행동 직결 (상태 기반 노출)

- 이중 이동 건너뛰기 버튼 (`PLAYER_DOUBLE_MOVE`에서만 표시)
- 능력 대상 선택 안내/오류
- 전투 종료 CTA(재시작/메인 메뉴)

## P2: 전황 판단 보조

- 아군/적군 기물 수
- 포획 수
- 위협도(낮음/보통/높음)

## P3: 장기 판단/분석 보조

- 추가 통계(연속 무손실 라운드, 최근 교환 비율 등) — v1에서는 미노출/후속
- 전투 로그 상세 패널 — 후속 이슈로 분리

## 5) 상태별 노출 규칙

| 상태 | 상단 상태바 | 보드 | 하단 액션바 | 오버레이/모달 |
|---|---|---|---|---|
| Empty (초기 데이터 없음) | P0 스켈레톤 | 비활성 | 비활성 | 재시도 CTA |
| Loading (전환 중) | P0 유지 + 로딩 인디케이터 | 상호작용 잠금 | 주요 버튼 잠금 | 없음 |
| Error (콘텐츠/런타임 오류) | 오류 요약(P0) | 잠금 또는 안전모드 | 복귀/재시도(P1) | 에러 다이얼로그 |
| Success (일반 전투) | P0 전체 | 활성 | 맥락 기반 버튼 | 필요 시 업그레이드/결과 모달 |

### 전환 정책

- `Success → Loading`: 업그레이드 화면 전환 직전 300ms 이내 버튼 중복 탭 방지 잠금.
- `Loading → Success`: 복귀 시 직전 P0 문맥(턴/목표) 복원.
- `Success → Error`: 보드 입력 즉시 중단 후 사용 가능한 복구 액션 제공.

## 6) 컴포넌트 분해 (DEV handoff)

## A. TopStatusBar (P0)

- 필드: `round`, `maxRounds`, `turnState`, `objectiveText`, `runProgress`
- 데이터 소스: `GameState.currentRound`, `maxRounds`, `turnState`, 파생 목표 문구
- 규칙: 어떤 상태에서도 높이 고정, 레이아웃 점프 금지

## B. BoardViewport (Core Interaction)

- 필드: `board`, `selectedPieceId`, `validMoves`, `interactionEnabled`
- 데이터 소스: `GameState.board`, `selectedPieceId`, `validMoves`
- 규칙: 선택 불가 상태(로딩/결과)에서 입력 잠금 처리

## C. BottomActionPanel (P1/P2)

- 필드: `playerPieces`, `enemyPieces`, `capturedByPlayer`, `showSkipDoubleMove`, `showEndGame`
- 데이터 소스: `GameState`, 보드 파생 통계
- 규칙: P1 버튼 우선 배치, P2 수치는 1줄 내 축약

## D. ResultModal (P1)

- 필드: `victory`, `score`, `awardedCurrency`
- 데이터 소스: `GameEvent.RunEnded`
- 규칙: 강제 확인 흐름(닫기 X), 다음 행동 2개만 노출

---

## 7) 인터랙션 우선순위 규칙

1. **탭 1순위**: 보드 선택/이동
2. **탭 2순위**: 맥락 액션(이중 이동 건너뛰기)
3. **탭 3순위**: 세션 종료(게임 종료)

충돌 시 정책:

- 능력 대상 선택 대기 중에는 보드 탭만 유효.
- 적 턴 중에는 행동 버튼 비활성 또는 숨김.
- 결과 모달 노출 중에는 모든 배경 입력 차단.

---

## 8) 접근성/로컬라이즈 가이드 (v1 필수)

- 텍스트 대비: 상태 강조색 대비 4.5:1 이상
- 의미 전달 중복: 색 + 텍스트 동시 사용(예: 위협도)
- 로컬라이즈 키 기반 문구 사용, 하드코딩 금지
- 버튼 최소 터치 영역 48dp 유지

---

## 9) Handoff Criteria (개발 전달 완료 조건)

아래 8개를 모두 만족하면 DEV-6 완료로 본다.

1. P0~P3 우선순위가 컴포넌트 단위로 문서화되어 있음
2. S1~S4 상태별 노출 규칙이 정의되어 있음
3. Empty/Loading/Error/Success 처리 정책이 정의되어 있음
4. `GameState`/`GameEvent` 연결 필드가 명시되어 있음
5. 입력 충돌 정책(능력 선택/적 턴/결과 모달)이 명시되어 있음
6. 접근성/로컬라이즈 최소 기준이 포함되어 있음
7. DEV-9, DEV-8, DEV-10 연계 포인트가 식별되어 있음
8. 남은 결정 사항과 오너가 명확히 기록되어 있음

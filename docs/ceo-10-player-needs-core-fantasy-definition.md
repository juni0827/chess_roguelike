# CEO-10: 체스 로그라이크 플레이어 니즈와 코어 판타지 정의

Date: 2026-04-18  
Owner: CEO Team (handoff 대상: DEV / DESIGN)

---

## Problem

현재 프로젝트는 전투/업그레이드 루프의 기본 뼈대는 존재하지만, 아래 항목이 명확히 합의되지 않아 하위 이슈(CEO-11, DEV-3/7/8/10)에서 기준이 흔들릴 수 있다.

- 누구를 1차 타깃으로 잡는지(체스 숙련자 중심인지, 로그라이크 빌드러 중심인지)
- 플레이어가 한 런에서 기대하는 감정 보상(코어 판타지) 우선순위
- "좋은 기능"을 판별하는 공통 평가 프레임

이로 인해 기능 우선순위가 팀/담당자마다 달라지고, 데모 범위에서 과설계 혹은 방향 불일치가 발생할 위험이 있다.

---

## User Story

### Archetype A — Tactical Improver (체스/전략 숙련 지향)

- **누가:** 짧은 시간 안에 전략적으로 성장하는 플레이를 선호하는 플레이어
- **무엇을:** 매 턴의 선택이 명확한 결과로 이어지는 전투와, 실수에서 배울 수 있는 피드백
- **왜:** "내 판단이 승패를 만든다"는 감각(Competence)을 원함

### Archetype B — Build Crafter (로그라이크 빌드 실험 지향)

- **누가:** 반복 플레이에서 매번 다른 조합 실험을 즐기는 플레이어
- **무엇을:** 기물/능력/유물의 시너지를 통해 런마다 다른 빌드 정체성 확보
- **왜:** "이번 판은 저번 판과 다르게 풀린다"는 감각(Autonomy)을 원함

### Archetype C — Dark Fantasy Explorer (무드/연출 몰입 지향)

- **누가:** 다크 판타지 톤, 긴장감 있는 전투 연출, 보상 순간의 카타르시스를 선호하는 플레이어
- **무엇을:** 전투 위험 신호가 읽히고, 승리/획득 순간이 강하게 보상되는 경험
- **왜:** 몰입감과 감정적 보상(Immersion + Relatedness to world)을 원함

### 통합 User Story

> 플레이어는 한 손에 들어오는 짧은 런 안에서, 체스식 전술 판단으로 위기를 풀고,
> 로그라이크식 빌드 선택으로 자신만의 전투 정체성을 만들며,
> 다크 판타지 연출 속에서 "내가 설계한 수"로 승리했다는 성취를 반복해서 얻고 싶다.

## Core Fantasy

### Primary Fantasy (1문장)

**"예측 가능한 위협 위에서, 내가 설계한 수와 빌드로 판을 뒤집는 다크 택티컬 체스 러너."**

### Supporting Fantasy Axes (3축)

1. **Tactical Mastery**  
   - 적 의도(Threat/Intent)가 읽히고, 대응 수를 찾아내는 재미
2. **Build Expression**  
   - 기물 성장 + 능력 + 패시브 조합으로 런 정체성을 만드는 재미
3. **Ritualized Payoff**  
   - 전투 승리, 희귀 보상 획득, 위험 감수 성공 순간의 강한 보상 연출

## Design Pillars (기능 판단 기준)

1. **읽히는 위협, 의미 있는 한 수**  
   - 플레이어는 "왜 위험한지"와 "무슨 대응이 가능한지"를 즉시 이해할 수 있어야 한다.
2. **매 런 다른 정체성**  
   - 보상 선택이 단순 수치 증가가 아닌 플레이 스타일 변화를 만들 것.
3. **짧지만 압축된 의사결정**  
   - 모바일 기준 1~2분 단위로 중요한 선택이 반복될 것.
4. **실패도 학습이 되는 구조**  
   - 패배 시 "무엇이 문제였는지" 복기 가능한 피드백을 제공할 것.
5. **보상 순간의 감정 증폭**  
   - 승리/획득/승급 순간은 시각·사운드·정보 구조가 명확히 강조될 것.

---

## Feature Evaluation Checklist (제안 기능 공용 체크)

새 기능을 추가할 때 아래 6문항 중 **최소 4개 이상**을 만족해야 채택 후보로 본다.

1. 적 의도/위험을 더 잘 읽게 만드는가?
2. 플레이어의 선택지 다양성을 실제로 늘리는가?
3. 기존 빌드 축(기물/능력/패시브)과 시너지가 있는가?
4. 1턴 혹은 1노드 내 즉시 체감 가능한가?
5. UI 인지 부담을 과도하게 늘리지 않는가?
6. 데모 범위(현재 사이클)에서 구현 가능성이 높은가?
